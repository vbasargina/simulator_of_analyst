На выходе будет DAG в airflow, который будет считаться каждый день за вчера. 

1. В feed_actions для каждого юзера считается число просмотров и лайков контента. В message_actions для каждого юзера считается, сколько он получает и отсылает сообщений, скольким людям он пишет, сколько людей пишут ему. Каждая выгрузка делается в отдельном таске.

2. Далее объединяется две таблицы в одну.

3. Для этой таблицы считаются все эти метрики в разрезе по полу, возрасту и ос. Получается три разных таска на каждый срез.

4. И финальные данные со всеми метриками записываются в отдельную таблицу в ClickHouse.

5. Каждый день таблица дополнятся новыми данными. 

Структура финальной таблицы:

Дата - event_date

Название среза - dimension

Значение среза - dimension_value

Число просмотров - views

Число лайков - likes

Число полученных сообщений - messages_received

Число отправленных сообщений - messages_sent

От скольких пользователей получили сообщения - users_received

Скольким пользователям отправили сообщение - users_sent

Срез — это os, gender и age
